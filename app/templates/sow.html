Statement of Work (SOW)

Introduction

This Statement of Work (SOW) outlines the contributions and responsibilities of each team member for the "Mobile App Development for Customer Service" project. Each role is critical to ensuring the project's success and timely completion.

Roles and Responsibilities

  • Project Manager (PM): Seth Williams

    The Project Manager is responsible for directing the work, managing communication, and ensuring the timely completion of assignments. Specific tasks include:

    • Scheduling team meetings and ensuring attendance.
    • Assigning tasks and tracking progress using tools such as the Gantt Chart.
    • Facilitating communication among team members to resolve issues promptly.
    • Submitting final deliverables with all team members' names included.

    Example Contribution: The PM ensures all tasks are on track and makes sure that all issues are being resolved.

  • Developer: Seth Williams

    The Developers are responsible for writing and maintaining the code for the portfolio. Specific tasks include:

    • Implementing interactive components such as the project calendar and Gantt Chart.
    • Ensuring functionality and addressing technical issues as they arise.
    • Testing features to ensure seamless integration and usability.

    Example Contribution: The Developer writes Python scripts to enable users to add, edit, and delete events in the project calendar.

  • Designer: Seth Williams

    The Designers focus on the visual aspects of the portfolio to ensure it is visually appealing and easy to navigate. Specific tasks include:

    • Designing user-friendly interfaces for all pages in the portfolio.
    • Ensuring the design meets accessibility standards (e.g., WCAG compliance).
    • Enhancing the user experience by improving layout and navigation.

    Example Contribution: The Designer creates an intuitive and attractive layout for the Gantt Chart page, ensuring it is accessible on all devices.

  • Researcher: Seth Williams

    The Researchers gather the necessary information to support the content and development of the portfolio. Specific tasks include:

    • Researching project management concepts, best practices, and tools.
    • Providing relevant data for sections such as Critical Path Analysis and Risk Management Plans.
    • Ensuring all content is accurate, well-referenced, and aligned with project goals.

    Example Contribution: The Researcher compiles resources on project network diagrams and provides information to the developers to implement.

Collaboration and Deliverables

  • The Project Manager ensures all team members collaborate effectively by facilitating weekly meetings and addressing any communication barriers.
  • Developers and Designers work closely to align functionality with the visual design of the portfolio.
  • Researchers provide valuable insights and data to support accurate and meaningful content across all portfolio components.
  • Deliverables include a functional and visually appealing portfolio, a completed Gantt Chart, a detailed Work Breakdown Structure, and comprehensive project documentation.

Summary

The success of the project depends on the collaboration and individual contributions of each team member. By fulfilling their respective responsibilities, the team will deliver a high-quality portfolio that meets the project's objectives and exceeds stakeholder expectations.
